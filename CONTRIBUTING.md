# Contributing to py-deep-dive 🐍

Thanks for wanting to help explain Python internals! Here's how:

## Ways to Contribute

- **New topics** — Open an issue first to discuss scope
- **Fix errors** — PRs welcome for any inaccuracies
- **Better diagrams** — ASCII art improvements are gold
- **More explore.py experiments** — The more "aha!" moments, the better

## Style Guide

- Write like you're explaining to a smart friend, not a textbook
- Include runnable code — no pseudo-code
- Add ASCII diagrams for memory layouts
- Use 🤯 for mind-blowing facts and 💡 for staff engineer insights
- Keep `explore.py` files dependency-free (stdlib only)

## Process

1. Fork the repo
2. Create a branch (`git checkout -b topic/your-topic`)
3. Write your content following existing topic structure
4. Test all `explore.py` scripts with Python 3.8+
5. Open a PR

## Topic Structure

```
NN-topic-name/
├── README.md      # Visual explanation with diagrams
└── explore.py     # Runnable script proving the concepts
```

Questions? Open an issue. Let's make Python internals accessible to everyone.
